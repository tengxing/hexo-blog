发表于 2018-04-29

前言

Sometimes我们需要为数据库运行类似类型的批量查询，例如将数据从CSV文件加载到关系数据库表中。 我们知道我们可以选择使用Statement或PreparedStatement来执行查询。 除此之外，JDBC还提供了Batch Processing功能，通过该功能，我们可以一次执行大量查询来处理数据库。下面我将通过不通维度来评估批处理性能。

评估

下面的测试结果仅仅是个参考,因为一个全面的测试不仅仅只是考虑这几个方面，如：机器性能，数据库类型等等

mysql STATEMENT PREPAREDSTATEMENT STATEMENT BATCH PREPAREDSTATEMENT BATCH
rewriteBatchedStatements=false 14666 14666 11605 12166
rewriteBatchedStatements=true 14792 13681 11755 248

说明:在mysql中，默认不会开启batch query，需要设置rewriteBatchedStatements=true，才能实现高性能的批量插入
