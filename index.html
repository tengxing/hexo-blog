后记

上面只是说明了一种常用的方式进行自启动，可以如果还尽早的进行脚本执行的话，可以根据需要放在rc*.d目录下。

  • 0~6表示启动优先级，数字越小优先级越高，优先启动。
  • 进入rc*.d目录你会发现，里面都是一个以Kxx或者Sxx开头的软链接
  • 带K的表示停止(Kill)一个服务，S表示开启(Start)的意思
  • 一般来说标准的shell脚本最少接受两个参数start和stop，它们分别代表启动和停止服务，额外的参数有:restart,reload,status,force-reload。
  • 当执行Kxx或者Sxx的脚本时候，会传入stop或start参数，来告诉Kxx链接到的脚本(/etc/init.d/xxx)是启动(/etc/rc.d/init.d/xxx start)或者是停止(/etc/rc.d/init.d/xxx stop)。

nginx使用ssl模块配置https.md

发表于 2018-05-31

前言

由于项目上用到小程序,而现在的小程序非常的严格，在开发的时候对接外部接口需要先添加审核，还得是https协议，而不是htpp协议，对于很多JAVA Web开发人员来说，服务端大多使用时下流行的tomcat(默认是http)，并且在虚拟服务器使用nginx作请求转发操作。因此，为了实现https访问服务器，我们可以配置nginx监听htpps的默认端口，再进行转发操作。

http与https

概念

  • HTTP：是互联网上应用最为广泛的一种网络协议，是一个客户端和服务器端请求和应答的标准（TCP），用于从WWW服务器传输超文本到本地浏览器的传输协议，它可以使浏览器更加高效，使网络传输减少。

  • HTTPS：是以安全为目标的HTTP通道，简单讲是HTTP的安全版，即HTTP下加入SSL层，HTTPS的安全基础是SSL，因此加密的详细内容就需要SSL。

　　HTTPS协议的主要作用可以分为两种：一种是建立一个信息安全通道，来保证数据传输的安全；另一种就是确认网站的真实性。

区别

  1. https协议需要到ca申请证书，一般免费证书较少，因而需要一定费用。
  2. http是超文本传输协议，信息是明文传输，https则是具有安全性的ssl加密传输协议。
  3. http和https使用的是完全不同的连接方式，用的端口也不一样，前者是80，后者是443。
  4. http的连接很简单，是无状态的；HTTPS协议是由SSL+HTTP协议构建的可进行加密传输、身份认证的网络协议，比http协议的明文传输安全。
  5. http在网络速度上略快于https，https的缓存不是很好，开销比http多
  6. https在通信前会有一个证书认证的过程，类似与websocket的第一次握手。

感觉除了安全之外,没有啥子优势(安全本身就是一个很大优势)

Java JDBC事务管理和保存点.md

发表于 2018-05-03

前言

当我们有一组要执行的查询时，事务管理非常重要，并且我们希望确保它们全部执行或者不执行它们。 我们可以将连接自动提交设置为false，以实现事务管理。 如果一切顺利，我们可以提交交易，或者如果有任何异常，我们可以回滚整个交易或者回滚到特定点。

正常情况下

默认情况下，当我们创建数据库连接时，它以auto-commit模式运行。 这意味着每当我们执行一个查询并且它完成时，提交就会自动触发。 因此，我们触发的每个SQL查询都是一个事务，如果我们正在运行一些DML或DDL查询，那么在每个SQL语句完成后，这些更改都会保存到数据库中。

事务全回滚

JDBC API提供了方法setAutoCommit（），通过它我们可以禁用连接的自动提交功能。 我们应该只在需要的时候禁用自动提交，因为除非我们在连接上调用commit（）方法，否则事务将不会被提交。 数据库服务器使用表锁实现事务管理并且是资源密集型过程。 所以我们应该尽快完成交易。 让我们编写另一个程序，我们将使用JDBC事务管理功能来确保数据完整性不被违反。

通过事物回滚就解决上面事务取消的问题，但是这个时候你可能回想，我不想全部回滚呢？答案是肯定的。

Savepoint

有时一个事务可以是多个语句的组合，我们希望回滚到事务中的某个特定点。
JDBC Savepoint帮助我们在事务中创建检查点，如果我们得到任何异常，我们可以选择回滚作为事务一部分执行的所有查询，回滚到特定的检查点。

基于Ubuntu16.04安装Docker.md

发表于 2018-05-03

前言

Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口。

Docker自2013年以来非常火热，无论是从 github 上的代码活跃度，还是Redhat在RHEL6.5中集成对Docker的支持, 就连 Google 的 Compute Engine 也支持 docker 在其之上运行。

卸载老版本

老版本的Docker被称为docker或docker-engine，Docker CE包现在称为docker-ce。 如果这些已安装，请将其卸载：

/var/lib/docker/的内容（包括镜像，容器等）将被保留

安装存储驱动

Ubuntu上的Docker CE支持overlay2和aufs存储驱动程序。
对于Ubuntu 16.04及更高版本，Linux内核包含对OverlayFS的支持，而Docker CE默认使用overlay2存储驱动程序。 如果您需要使用aufs，则需要手动配置它。
