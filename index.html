基于Ubuntu16.04安装Docker.md

发表于 2018-05-03

前言

Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口。

Docker自2013年以来非常火热，无论是从 github 上的代码活跃度，还是Redhat在RHEL6.5中集成对Docker的支持, 就连 Google 的 Compute Engine 也支持 docker 在其之上运行。

卸载老版本

老版本的Docker被称为docker或docker-engine，Docker CE包现在称为docker-ce。 如果这些已安装，请将其卸载：

/var/lib/docker/的内容（包括镜像，容器等）将被保留

安装存储驱动

Ubuntu上的Docker CE支持overlay2和aufs存储驱动程序。
对于Ubuntu 16.04及更高版本，Linux内核包含对OverlayFS的支持，而Docker CE默认使用overlay2存储驱动程序。 如果您需要使用aufs，则需要手动配置它。

安装Docker CE

可以根据需要以不同的方式安装DockerCE：

  • 源安装，以便安装和升级任务。 推荐这种方法。

  • DEB软件包并手动安装，并完全手动管理升级。

  • 脚本来安装Docker。

当一个container起来之后，我们有时候希望能进入container内部去看看，比如查查日志，执行些操作等。目前有几种方式可以实现：

  • docker attach
  • lxc-attach
  • nsenter
  • ssh

docker attach
